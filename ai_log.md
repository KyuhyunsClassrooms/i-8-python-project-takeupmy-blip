# Gemini 활용 기록

> 이번 수행평가에서는 학교 계정 Gemini 사용을 허용합니다.  
> 단, Gemini가 만든 코드를 이해하지 못한 채 그대로 제출하면 안 됩니다.  
> 실제 개인정보, 친구 이름, 성적, 전화번호, 주소 등은 절대 입력하지 않습니다.

---

## 1차 Gemini 활용

- 사용 날짜:6월 11일
- 사용 목적:계획서 작성
- 내가 입력한 프롬프트:
```text
입력 → 처리 → 출력 구조에 맞게 사용하는 법
2차원 리스트를 주제에 맞게 사용하는 법
함수를 기능별로 나누어 작성하는 법
```

- Gemini가 제안한 내용 요약:
- 내가 반영한 부분:입력 출력 처리구조, 2차원 리스트 만들기
- 내가 수정하거나 사용하지 않은 부분:주제 선정
- 반영 위치: 예) project_plan.md 4번, main.py의 data 구조
입력 출력 처리, 2차원 리스트

## 2차 Gemini 활용

- 사용 날짜:6월 12일
- 사용 목적:메인 코드 작성
- 내가 입력한 프롬프트:

```
이번 수행평가는 파이썬으로 나만의 자유 주제 프로그램을 만드는 프로젝트입니다.
단순히 코드가 실행되는지만 평가하지 않습니다. 다음을 함께 봅니다.
내가 해결하려는 문제를 잘게 나누어 설명했는가?
입력 → 처리 → 출력 구조가 명확한가?
2차원 리스트를 주제에 맞게 사용했는가?
함수를 기능별로 나누어 작성했는가?
조건문과 반복문을 이용해 실제 판단, 검색, 계산, 추천, 판정 등이 이루어지는가?
학교 계정 Gemini를 활용하되, 최종 코드를 본인이 이해하고 설명할 수 있는가?
보고서에 설계 과정, 코드 설명, 실행 결과, 개선 방안을 정리했는가?
코드인터뷰에서 자신의 코드를 설명할 수 있는가? 
```

- Gemini가 제안한 내용 요약:초기 제안받은 카테고리별 퀴즈 주제를 '과학 상식 퀴즈(쉬움/보통 난이도 선택)' 구조로 직접 발전. Gemini에게 지금까지의 설계 및 대화 내용을 요약받아 프로젝트 진행 상황을 점검
- 내가 반영한 부분:코드 작성
- 내가 수정하거나 사용하지 않은 부분:제미나이가 제안한 문제 내용, 과목 주제로 나누어서 상식퀴즈 만들기
- 반영 위치:main.py

## 3차 Gemini 활용

- 사용 날짜:6월 16일
- 사용 목적:내용 요약
- 내가 입력한 프롬프트:

```text
내가 지금까지 너한테 물어본거 요약해줘
```

- Gemini가 제안한 내용 요약:
- 내가 반영한 부분:초기 제안받은 카테고리별 퀴즈 주제를 '과학 상식 퀴즈(쉬움/보통 난이도 선택)' 구조로 직접 발전시켰음. Gemini에게 지금까지의 설계 및 대화 내용을 요약받아 프로젝트 진행 상황을 점검함
- 내가 수정하거나 사용하지 않은 부분:
- 반영 위치:2차 제미나이가 제안한 내용 요약

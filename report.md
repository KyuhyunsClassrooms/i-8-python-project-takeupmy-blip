## 2. 문제 분해 및 모델링

### 입력
-사용자가 선택한 과학 퀴즈의 난이도 

-사용자가 각 문제마다 입력하는 퀴즈의 정답

### 처리
-사용자가 입력한 난이도와 2차원 리스트 quiz_show의 0번 열을 비교하여 일치하는 문제만 추출함
-반복문(for)을 사용하여 해당 난이도의 문제를 순차적으로 화면에 출제함.
-조건문(if)을 사용하여 사용자의 입력값과 2차원 리스트의 2번 열(실제 정답)을 비교하여 채점하고 점수(score)를 누적함.

### 출력
-매 문제마다 정답 여부 알림
-모든 문제가 끝난 후 최종 결과 출력

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:

한 행의 의미:

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 퀴즈의 출제 난이도 | "쉬움", "보통" |
| 1번 열 | 사용자에게 보여줄 퀴즈 질문 | "H2O는 어떤 물질인가", "유전자 정보를 가지고 있는 것은 무엇인가" |
| 2번 열 | 사용자의 입력과 비교할 실제 정답 | "물", "DNA" |
| 3번 열 | 사용하지 않음 | 없음 |

이 2차원 리스트를 사용한 이유:여러 개의 퀴즈 데이터를 하나의 변수로 과리하기 위해서

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| show_difficulty_menu | 난이도 메뉴를 보여주고 사용자의 선택을 입력받음 | 없음 | 사용자가 고른 난이도 |
| run_quiz | 선택된 난이도의 문제를 출제하고 채점함 | selected_difficulty (선택된 난이도) | 맞힌 개수 (score, 정수), 총 문제 수 (total, 정수) |
| print_result | 최종 맞힌 개수와 결과를 화면에 출력함 | score (맞힌 개수), total (총 문제 수) | 없음 (화면 출력) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:run_quiz(selected_difficulty)

설명:이 함수는 핵심 로직을 담당하는데 2차원 리스트를 활용해 사용자가 고른 난이도에 맞는 문제를 추제하고 최종적으로 총 문제 수와 정답 수를 나타낸다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
-show_difficulty_menu 함수에서 사용자가 입력한 번호가 "쉬움"인지 "보통"인지 확인하기 위해 사용했습니다.
-run_quiz 함수에서 2차원 리스트 속 문제의 난이도가 사용자가 고른 난이도와 같은지(quiz[0] == selected_difficulty) 검사했습니다.
-사용자가 입력한 답이 실제 정답과 일치하는지(user_answer == quiz[2]) 검사했습니다.

### 반복문 사용 부분

무엇을 반복 처리했는가?
-2차원 리스트 quiz_show에 저장된 전체 문제 데이터(행)를 하나씩 순서대로 꺼내와서 검사하는 과정을 반복 처리했습니다.


### 조건문과 반복문이 함께 작동한 부분
-run_quiz 함수 내부에서 for문을 통해 quiz_show 리스트의 데이터를 하나씩 확인하는 동시에, 내부의 if문들을 통해 특정 난이도의 문제만 걸러내어 화면에 출력하고 사용자의 정답 여부를 판정하는 로직이 함께 작동했습니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:난이도 입력에 "보통", 정답 입력에 차례대로 "운동에너지,위치에너지", "4가지", "조직계", "산성" 입력

예상 결과:보통 난이도의 4문제가 모두 출제되고, "총 4문제 중 4문제를 맞혔습니다!" 출력

실제 결과:예상했던 결과와 일치하게 정상적으로 작동함.


### 테스트 2: 예외 또는 경계 사례

입력:난이도 선택 시 "쉬움"이나 "보통"이 아닌 "어려움" 또는 오타를 입력했을 때

예상 결과:difficulty 변수가 빈 문자열("")로 반환되거나 프로그램이 정상적으로 문제를 출제하지 못해야 함.

실제 결과:아무런 문제도 출제되지 않고 바로 게임 종료 화면이 나옴. (향후 올바른 값을 입력할 때까지 반복해서 물어보는 기능이 필요함을 인지함)


### 발견한 오류와 수정한 방법
-2차원 리스트 문법 오류: quiz_show 리스트 안에서 각 행(소리스트)을 구분할 때 쉼표(,)를 빠뜨려 SyntaxError가 발생했습니다. 각 행의 대괄호 끝에 쉼표를 추가하여 해결했습니다.
-함수 들여쓰기 및 오타 오류: show_difficulty_menu 함수 내부의 코드가 들여쓰기가 안 되어 함수 외부 코드로 인식되었고, elif를 elie로 잘못 입력하여 문법 오류가 났습니다. 코드를 함수 안으로 들여쓰고 키워드를 elif로 올바르게 고쳤습니다. 또한 결괏값을 전달하기 위해 return문을 추가했습니다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:
아이디어를 구체화하여 '난이도 선택형 과학 상식 퀴즈'라는 주제와 2차원 리스트 구조 설계를 제안받았습니다. 또한 코드 작성 중 발생한 들여쓰기 에러와 쉼표 누락 등의 문법 오류 원인을 명확하게 설명 받아 해결했습니다.

Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:Gemini는 처음에 번호를 선택하면 카테고리(과학, 역사 등)를 반환하는 방식을 제안했으나, 제가 관심 있는 과학 과목에 집중하고 싶어서 '난이도(쉬움/보통)'를 선택하는 구조로 변경하여 적용했습니다.


최종 코드에서 내가 직접 판단하거나 수정한 부분:
과학 교과서 내용을 기반으로 2차원 리스트에 들어갈 실제 문제 내용과 정답 데이터("역학적 에너지", "조직계", "PH" 등)를 직접 고르고 구성했습니다.

## 9. 한계와 개선 방안

현재 프로그램의 한계:사용자가 난이도를 고를 때 "쉬움"이나 "보통"이 아닌 잘못된 문자열을 입력하면, 다시 입력할 기회를 주지 않고 빈 화면의 결과만 출력한 채 프로그램이 종료되는 한계가 있습니다.


다시 개선한다면 추가하고 싶은 기능:while 반복문을 활용하여 사용자가 난이도 메뉴를 잘못 입력했을 때 "올바른 난이도를 다시 입력하세요"라고 메시지를 띄우며 무한히 다시 입력받는 예외 처리 기능을 추가하고 싶습니다.
